12.1 Learning Objectives

By the end of this chapter you will be able to:

  • Understand sources of variability in PK parameters
  • Distinguish fixed effects from random effects
  • Build a population PK model with covariates
  • Use nlmixr2 for nonlinear mixed-effects modeling
  • Interpret population parameters, individual predictions, and shrinkage

12.2 Why Population PK?

In classical PK, we analyze one subject at a time (individual PK). But in drug development, we need to:

  1. Analyze sparse data (1–3 samples per subject from hundreds of patients)
  2. Quantify between-subject variability (why does CL vary 5-fold?)
  3. Identify covariates (weight, age, renal function) that explain variability
  4. Support dose individualization

Population PK (PopPK) uses nonlinear mixed-effects models to analyze all subjects simultaneously.

12.3 The Population Model Structure

12.3.1 Hierarchical Model

Level 1 — Structural (PK) model: C_{ij} = f(t_{ij}, \theta_i) + \varepsilon_{ij}

Level 2 — Between-subject variability (BSV): \theta_i = \theta_{pop} \cdot e^{\eta_i}, \quad \eta_i \sim N(0, \omega^2)

Level 3 — Covariate effects: \theta_i = \theta_{pop} \cdot \left(\frac{WT_i}{70}\right)^{\beta_{WT}} \cdot e^{\eta_i}

Level 4 — Residual error: \varepsilon_{ij} \sim N(0, \sigma^2) \quad \text{or} \quad C_{ij} = f(t_{ij}, \theta_i) \cdot (1 + \varepsilon_{ij})

Code 
library(ggplot2)
library(dplyr)

# Conceptual visualization: Between-subject variability
set.seed(123)
n_subj <- 50
times <- seq(0, 24, by = 0.5)

# Generate individual PK profiles with variability in CL and V
pop_CL <- 5     # L/h
pop_V  <- 50    # L

# Log-normal BSV: ω_CL = 0.3, ω_V = 0.2
set.seed(42)

sim_pop <- expand.grid(time = times, ID = 1:n_subj) %>%
  mutate(
    eta_CL = rep(rnorm(n_subj, 0, 0.3), each = length(times)),
    eta_V  = rep(rnorm(n_subj, 0, 0.2), each = length(times)),
    CL_ind = pop_CL * exp(eta_CL),
    V_ind  = pop_V * exp(eta_V),
    ke_ind = CL_ind / V_ind,
    dose   = 500,
    conc   = (dose / V_ind) * exp(-ke_ind * time),
    conc_obs = conc * exp(rnorm(n(), 0, 0.1))
  )

# Plot individual profiles
ggplot(sim_pop, aes(time, conc, group = ID)) +
  geom_line(alpha = 0.3, color = "#0B6E6B") +
  labs(title = "Population PK: 50 Simulated Subjects",
       subtitle = "Between-subject variability in CL (ω=0.3) and V (ω=0.2)",

12.4 Population PK with nlmixr2

Code 
library(nlmixr2)

# Define a one-compartment model for nlmixr2
pop_model <- function() {
  ini({
    # Population parameters (fixed effects)
    tka <- log(1.0)     # log ka
    tcl <- log(5.0)     # log CL
    tv  <- log(50)      # log V

    # Between-subject variability (random effects)
    eta.ka ~ 0.3
    eta.cl ~ 0.3
    eta.v  ~ 0.2

    # Residual error
    add.err <- 1
    prop.err <- 0.1
  })
  model({
    ka <- exp(tka + eta.ka)
    cl <- exp(tcl + eta.cl)
    v  <- exp(tv + eta.v)

    d/dt(depot)   <- -ka * depot
    d/dt(central) <-  ka * depot - (cl/v) * central
    cp <- central / v
    cp ~ add(add.err) + prop(prop.err)
  })
}

12.7 Model Diagnostics for Population PK

12.7.1 Key Diagnostics

  1. Observed vs Predicted: Individual predictions (IPRED) should follow identity
  2. Residuals vs Predicted: No trends, symmetric around zero
  3. Distribution of random effects (\eta): Should be approximately normal
  4. Shrinkage: High shrinkage (\eta-shrinkage > 20–30%) reduces reliability of individual estimates
Code 
# Simulate diagnostic plots
set.seed(456)
n <- 200
pred <- runif(n, 0.1, 50)
obs <- pred * exp(rnorm(n, 0, 0.2))

diag_df <- data.frame(
  PRED = pred,
  IPRED = obs,
  CWRES = (log(obs) - log(pred)) / 0.2
)

# Observed vs Predicted
p1 <- ggplot(diag_df, aes(PRED, IPRED)) +
  geom_point(alpha = 0.5, color = "#0B6E6B") +
  geom_abline(slope = 1, intercept = 0, color = "#C44536", linetype = "dashed") +
  labs(title = "Observed vs Population Predicted", x = "PRED", y = "DV") +

# Residuals vs Predicted

12.8 Shrinkage

Shrinkage occurs when individual data are sparse and individual estimates “shrink” toward the population mean:

\eta\text{-shrinkage} = 1 - \frac{var(\hat{\eta}_i)}{\omega^2}

  • Low shrinkage (< 10%): Individual estimates are reliable
  • Moderate shrinkage (10–30%): Caution needed for covariate selection using EBEs
  • High shrinkage (> 30%): Individual estimates unreliable; use population predictions

12.8.1 Exercises

  1. Simulate a population PK dataset with 200 subjects, 3 samples each. Fit using nlmixr2 and report \omega_{CL} and \omega_V.
  2. Add body weight as a covariate on CL. Does it reduce \omega_{CL}? By how much?
  3. Generate diagnostic plots for a fitted model. What patterns would indicate model misspecification?
  4. Explain why shrinkage matters for covariate model building.
  5. Compare rich vs sparse sampling designs: which gives better estimates of \omega^2?

12.8.2 Common Mistakes

  • Using individual PK methods on sparse data: Leads to biased, imprecise estimates
  • Ignoring shrinkage: EBEs with high shrinkage should not be used for covariate screening
  • Overfitting covariates: Each covariate consumes degrees of freedom; use scientific rationale
  • Assuming log-normal BSV for all parameters: check ETA distributions
  • Not checking parameter correlations: High correlations can destabilize the model

12.8.3 Key Takeaway

Population PK uses mixed-effects models to analyze all subjects simultaneously, quantifying both population-typical parameters and between-subject variability. Covariates like weight and renal function explain variability and enable dose individualization. Always check diagnostics and shrinkage before interpreting individual estimates.

12.9 Alternative Estimation Methods: SAEM and nlme

nlmixr2’s FOCE is excellent, but other algorithms have strengths for certain problems. Let’s compare SAEM (Stochastic Approximation Expectation-Maximization, via saemix) and the classic nlme package on a simulated PopPK dataset.

12.9.1 Simulated PopPK Dataset

Code 
library(nlmixr2)
library(saemix)
library(nlme)

# Simulate 1-cmt oral PopPK: 50 subjects, sparse sampling
set.seed(1234)
n_subj <- 50
pop_pk <- data.frame()
for (i in 1:n_subj) {
  CL_i  <- 5  * exp(rnorm(1, 0, 0.3))   # BSV 30%
  V_i   <- 80 * exp(rnorm(1, 0, 0.25))  # BSV 25%
  ka_i  <- 1.2
  t_i   <- sort(c(0.5, 1, 2, 3, 4, 6, 8, 12, 24))
  conc_i <- (100 / V_i) * (ka_i / (ka_i - CL_i/V_i)) *
    (exp(-(CL_i/V_i) * t_i) - exp(-ka_i * t_i)) +
    rnorm(length(t_i), 0, 0.5)
  pop_pk <- rbind(pop_pk,
    data.frame(ID = i, TIME = t_i, DV = pmax(conc_i, 0.01),
               AMT = 0, EVID = 0))
  pop_pk <- rbind(pop_pk,
    data.frame(ID = i, TIME = 0, DV = 0, AMT = 100, EVID = 1))
}
pop_pk <- pop_pk[order(pop_pk$ID, pop_pk$TIME), ]

# ---- nlmixr2 (FOCE) ----
fit_nlmixr <- tryCatch({
  mod_nlmixr <- nlmixr2({
    ka     <- exp(tka + eta.ka)
    cl     <- exp(tcl + eta.cl)
    v      <- exp(tv  + eta.v)
    d/dt(A1) <- -ka * A1
    d/dt(A2) <-  ka * A1 - (cl/v) * A2
    cp <- A2 / v
    cp ~ add(add.sd)
  })
  nlmixr(mod_nlmixr, pop_pk, est = "focei",
         control = foceiControl(maxOuterIterations = 0))
}, error = function(e) {
  message("nlmixr2 FOCE fitting failed: ", e$message)
  NULL
})

# ---- saemix ----
fit_saem <- tryCatch({
  saemix_data <- saemixData(
    name.data = pop_pk[pop_pk$EVID == 0, c("ID", "TIME", "DV")],
    name.group = "ID", name.predictors = "TIME",
    name.response = "DV"
  )
  
  model_saemix <- function(psi, id, xidep) {
    CL <- psi[id, 1]; V <- psi[id, 2]; ka <- psi[id, 3]
    (100 / V) * (ka / (ka - CL/V)) * (exp(-(CL/V) * xidep) - exp(-ka * xidep))
  }
  
  fit_saemix <- saemixModel(
    model = model_saemix,
    psi0  = c(CL = 5, V = 80, ka = 1.2),
    omega.init = diag(c(0.09, 0.06, 0.01)),
    error.model = "constant"
  )
  
  saemix(fit_saemix, saemix_data,
         saemix.options(seed = 123, displayProgress = FALSE))
}, error = function(e) NULL)

```{r}
pk_sparse <- expand.grid(ID = 1:n_subjects, time = sparse_times) %>%
  arrange(ID, time) %>%
  mutate(
    eta_CL = rep(rnorm(n_subjects, 0, 0.3), each = length(sparse_times)),
    eta_V  = rep(rnorm(n_subjects, 0, 0.2), each = length(sparse_times)),
    CL = 5 * exp(eta_CL),
    V  = 50 * exp(eta_V),
    ke = CL / V,
    DOSE = 500,
    conc = (DOSE / V) * exp(-ke * time),
    conc_obs = conc * exp(rnorm(n(), 0, 0.15)),  # 15% CV residual error
    CENS = 0  # Not censored
  ) %>%
  filter(conc_obs > 0.01)  # Remove BLOQ

head(pk_sparse, 10)
```

Each subject appears at 4 sparse time points (typical of Phase II/III trials where intensive sampling is infeasible). With 100 subjects but only 4 samples each, traditional individual fitting would fail — this is exactly why population PK pools information across subjects, "borrowing strength" to estimate both population-typical parameters and between-subject variability.

## Model Diagnostics for Population PK

### Key Diagnostics

1. **Observed vs Predicted**: Individual predictions (IPRED) should follow identity
2. **Residuals vs Predicted**: No trends, symmetric around zero
3. **Distribution of random effects ($\eta$)**: Should be approximately normal
4. **Shrinkage**: High shrinkage ($\eta$-shrinkage > 20–30%) reduces reliability of individual estimates

## Shrinkage

Shrinkage occurs when individual data are sparse and individual estimates "shrink" toward the population mean:

$$\eta\text{-shrinkage} = 1 - \frac{var(\hat{\eta}_i)}{\omega^2}$$

- **Low shrinkage (< 10%)**: Individual estimates are reliable
- **Moderate shrinkage (10–30%)**: Caution needed for covariate selection using EBEs
- **High shrinkage (> 30%)**: Individual estimates unreliable; use population predictions

::: {.mistakes-box}
### Common Mistakes

- **Using individual PK methods on sparse data**: Leads to biased, imprecise estimates
- **Ignoring shrinkage**: EBEs with high shrinkage should not be used for covariate screening
- **Overfitting covariates**: Each covariate consumes degrees of freedom; use scientific rationale
- **Assuming log-normal BSV for all parameters**: check ETA distributions
- **Not checking parameter correlations**: High correlations can destabilize the model
:::

::: {.key-box}
### Key Takeaway

Population PK uses mixed-effects models to analyze all subjects simultaneously, quantifying both population-typical parameters and between-subject variability. Covariates like weight and renal function explain variability and enable dose individualization. Always check diagnostics and shrinkage before interpreting individual estimates.
:::

## Alternative Estimation Methods: SAEM and nlme

nlmixr2's FOCE is excellent, but other algorithms have strengths for certain problems. Let's compare SAEM (Stochastic Approximation Expectation-Maximization, via `saemix`) and the classic `nlme` package on a simulated PopPK dataset.

### Simulated PopPK Dataset

```{r}
#| eval: true
#| label: fig-popest-comparison
#| fig-cap: "Individual PK fits from three estimation methods. SAEM (teal) and FOCE (coral) produce nearly identical population predictions, while nlme (purple) shows slight bias due to its first-order approximation."
#| fig-width: 9
#| fig-height: 6


library(nlmixr2)
library(saemix)
library(nlme)

# Simulate 1-cmt oral PopPK: 50 subjects, sparse sampling
set.seed(1234)
n_subj <- 50
pop_pk <- data.frame()
for (i in 1:n_subj) {
  CL_i  <- 5  * exp(rnorm(1, 0, 0.3))   # BSV 30%
  V_i   <- 80 * exp(rnorm(1, 0, 0.25))  # BSV 25%
  ka_i  <- 1.2
  t_i   <- sort(c(0.5, 1, 2, 3, 4, 6, 8, 12, 24))
  conc_i <- (100 / V_i) * (ka_i / (ka_i - CL_i/V_i)) *
    (exp(-(CL_i/V_i) * t_i) - exp(-ka_i * t_i)) +
    rnorm(length(t_i), 0, 0.5)
  pop_pk <- rbind(pop_pk,
    data.frame(ID = i, TIME = t_i, DV = pmax(conc_i, 0.01),
               AMT = 0, EVID = 0))
  pop_pk <- rbind(pop_pk,
    data.frame(ID = i, TIME = 0, DV = 0, AMT = 100, EVID = 1))
}
pop_pk <- pop_pk[order(pop_pk$ID, pop_pk$TIME), ]

# ---- nlmixr2 (FOCE) ----
fit_nlmixr <- tryCatch({
  mod_nlmixr <- nlmixr2({
    ka     <- exp(tka + eta.ka)
    cl     <- exp(tcl + eta.cl)
    v      <- exp(tv  + eta.v)
    d/dt(A1) <- -ka * A1
    d/dt(A2) <-  ka * A1 - (cl/v) * A2
    cp <- A2 / v
    cp ~ add(add.sd)
  })
  nlmixr(mod_nlmixr, pop_pk, est = "focei",
         control = foceiControl(maxOuterIterations = 0))
}, error = function(e) {
  message("nlmixr2 FOCE fitting failed: ", e$message)
  NULL
})

# ---- saemix ----
fit_saem <- tryCatch({
  saemix_data <- saemixData(
    name.data = pop_pk[pop_pk$EVID == 0, c("ID", "TIME", "DV")],
    name.group = "ID", name.predictors = "TIME",
    name.response = "DV"
  )
  
  model_saemix <- function(psi, id, xidep) {
    CL <- psi[id, 1]; V <- psi[id, 2]; ka <- psi[id, 3]
    (100 / V) * (ka / (ka - CL/V)) * (exp(-(CL/V) * xidep) - exp(-ka * xidep))
  }
  
  fit_saemix <- saemixModel(
    model = model_saemix,
    psi0  = c(CL = 5, V = 80, ka = 1.2),
    omega.init = diag(c(0.09, 0.06, 0.01)),
    error.model = "constant"
  )
  
  saemix(fit_saemix, saemix_data,
         saemix.options(seed = 123, displayProgress = FALSE))
}, error = function(e) NULL)

# ---- nlme ----
fit_nlme <- tryCatch({
  lme_model <- function(CL, V, ka, TIME) {
    (100 / V) * (ka / (ka - CL/V)) * (exp(-(CL/V) * TIME) - exp(-ka * TIME))
  }
  
  nlme(DV ~ lme_model(CL, V, ka, TIME),
       data   = pop_pk[pop_pk$EVID == 0, ],
       fixed  = CL + V + ka ~ 1,
       random = CL + V ~ 1 | ID,
       start  = c(CL = 5, V = 80, ka = 1.2),
       method = "ML")
}, error = function(e) NULL)

# ---- Parameter comparison table ----
est_table <- data.frame(
  Parameter = c("CL (L/h)", "V (L)", "ka (1/h)"),
  Truth     = c(5.0, 80, 1.2),
  FOCE      = if (!is.null(fit_nlmixr)) round(fixef(fit_nlmixr)["tcl"] |> exp(), 1) else NA,
  SAEM      = if (!is.null(fit_saem)) round(fit_saem@results@fixed.effects[1:3], 1) else NA,
  nlme      = if (!is.null(fit_nlme)) round(fixef(fit_nlme), 1) else NA
)
knitr::kable(est_table, caption = "Parameter estimates from 3 PopPK methods")
```

When all three methods succeed, compare the estimates against the truth column. FOCE typically recovers parameters closest to the true values under rich sampling, while SAEM excels with sparse data. The `nlme` package (first-order approximation) may show slight bias — this is expected because it approximates the likelihood rather than integrating over the random effects.

| Method | Algorithm | Speed | Handles ODEs | Best For |
|---|---|---|---|---|
| **FOCE (nlmixr2)** | Linearized likelihood | Medium | Yes | Rich data, complex models |
| **SAEM (saemix)** | Stochastic EM | Fast | Yes | Sparse data, many random effects |
| **nlme** | First-order/adaptive | Very fast | No (need closed-form) | Simple models, exploratory |
