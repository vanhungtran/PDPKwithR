4.4 Plotting Concentration-Time Profiles

4.4.1 Individual Profiles

Code 
# Individual concentration-time plots
ggplot(pk_data, aes(TIME, CONC, group = ID, color = ID)) +
  geom_point(size = 2) +
  geom_line(linewidth = 0.8) +
  labs(title = "Individual Concentration-Time Profiles",
       subtitle = "IV Bolus, 100 mg dose",
       x = "Time (hours)", y = "Concentration (mg/L)",
       color = "Subject") +
  theme_minimal()

Each colored line traces one subject’s concentration over 24 hours. The consistent exponential decay pattern across subjects confirms the IV bolus model, while the vertical spread at each time point reveals between-subject variability — a key concept that population PK models (Chapter 12) will formally quantify.

4.4.2 Mean ± SD Profile

Code 
# Summary statistics and mean ± SD plot
pk_summary <- pk_data %>%
  summarise(
    Mean = mean(CONC),
    SD   = sd(CONC),
    N    = n(),
    .by = TIME
  )

ggplot(pk_summary, aes(TIME, Mean)) +
  geom_ribbon(aes(ymin = Mean - SD, ymax = Mean + SD),
              fill = "#0B6E6B", alpha = 0.2) +
  geom_point(color = "#0B6E6B", size = 2) +
  geom_line(color = "#0B6E6B", linewidth = 1) +
  labs(title = "Mean Concentration-Time Profile (± SD)",
       x = "Time (hours)", y = "Concentration (mg/L)") +
  theme_minimal()

The ribbon shows ±1 SD around the mean at each time point, summarizing the population trend. The SD narrows at later times because concentrations are lower — this is multiplicative (log-normal) variability, not additive. Semi-log plots (below) make this pattern easier to assess.

4.4.3 Semi-Log Plot (Standard PK View)

Code 
ggplot(pk_data, aes(TIME, CONC, group = ID, color = ID)) +
  geom_point(size = 2) +
  geom_line(linewidth = 0.8) +
  scale_y_log10() +
  labs(title = "Semi-Log Concentration-Time Profiles",
       x = "Time (hours)", y = "Concentration (mg/L, log scale)",
       color = "Subject") +
  theme_minimal()

On the log scale, the exponential decay becomes a straight line — a quick visual check for first-order elimination. Parallel lines across subjects indicate similar elimination rates but different volumes of distribution (the vertical offset). This semi-log view is the standard PK diagnostic: deviations from linearity suggest multi-compartment behavior or nonlinear kinetics.

4.5 Plotting Dose-Response Data

Code 
# Simulate dose-response data
doses <- c(0.1, 0.3, 1, 3, 10, 30, 100, 300)
n_per_dose <- 5

set.seed(456)
dr_data <- expand.grid(DOSE = doses, REP = 1:n_per_dose) %>%
  mutate(
    Emax = 100,
    EC50 = 10,
    EFFECT = (Emax * DOSE) / (EC50 + DOSE) + rnorm(n(), 0, 5)
  )

ggplot(dr_data, aes(DOSE, EFFECT)) +
  geom_point(color = "#C44536", alpha = 0.6, size = 2) +
  scale_x_log10() +
  labs(title = "Dose-Response Data",
       x = "Dose (mg, log scale)", y = "Effect (% of maximum)") +
  theme_minimal()

The log-scale x-axis spreads low doses for closer inspection while compressing high doses where the effect saturates. The sigmoid shape is characteristic of receptor-mediated pharmacology: a threshold at low doses, a steep linear region around the EC_{50}, and a plateau as receptors become saturated.

4.6 Reproducible Scripts and Version Control

4.6.1 Best Practices for PK/PD Scripts

  1. Start every script with a header describing purpose, author, and date
  2. Load all packages at the top
  3. Use relative paths (not absolute paths like C:/Users/...)
  4. Comment your code — explain why, not what
  5. Separate data import, analysis, and visualization
  6. Use version control (Git) for all scripts

4.8 Handling Missing Data in PK/PD

Missing concentration values are common in clinical PK studies. The appropriate handling strategy depends on why the data are missing.

4.8.1 The MCAR/MAR/MNAR Framework

Mechanism Definition Test/Clue Action
MCAR (Missing Completely at Random) Missingness unrelated to any variable No pattern in missing vs observed Mean/median imputation OK
MAR (Missing at Random) Missingness explained by observed covariates Pattern linked to recorded variables (e.g., more missing at later times) Regression imputation, multiple imputation (mice), or MLE-based modeling
MNAR (Missing Not at Random) Missingness depends on the unobserved value itself (e.g., samples below LLOQ) Plausible mechanism: high CL patients drop out, BLQ samples Selection/pattern-mixture models + sensitivity analysis; no method is unbiased
Code 
# Example: Identify and handle missing data patterns
library(dplyr)

# Simulate PK data with different missingness types
set.seed(42)
pk_missing <- data.frame(
  ID = rep(1:10, each = 8),
  TIME = rep(c(0.5, 1, 2, 4, 6, 8, 12, 24), 10),
  CONC = 25 * exp(-0.2 * rep(c(0.5, 1, 2, 4, 6, 8, 12, 24), 10)) * 
          exp(rnorm(80, 0, 0.12))
)

# Introduce MCAR: random 10% missing
mcar_idx <- sample(1:80, 8)
pk_missing$CONC_MCAR <- pk_missing$CONC
pk_missing$CONC_MCAR[mcar_idx] <- NA

# Introduce MAR: more missing at later time points
mar_prob <- 0.1 * pk_missing$TIME / 24
mar_idx <- which(runif(80) < mar_prob)
pk_missing$CONC_MAR <- pk_missing$CONC
pk_missing$CONC_MAR[mar_idx] <- NA

# Summarize missingness
missing_summary <- pk_missing %>%
  summarise(
    MCAR_missing = sum(is.na(CONC_MCAR)),
    MAR_missing  = sum(is.na(CONC_MAR)),
    .by = TIME
  )

cat("Missing by time point:\n")

4.8.2 BLQ (Below Limit of Quantification) Handling

BLQ values are a special case of MNAR — the value is missing because it’s low. Common approaches:

  • LLOQ/2 substitution — set BLQ = LLOQ/2 (simple, common in NCA)
  • M3 method — likelihood-based approach that treats BLQ as left-censored observations
  • Imputation — multiple imputation with left-censoring constraint

For NCA, LLOQ/2 is standard. For population PK, likelihood-based methods (M3) are preferred as they correctly propagate uncertainty.

4.9 Beyond rxode2: Comparing Simulation Engines

While rxode2 is our workhorse for ODE-based PK modeling, R offers several complementary simulation frameworks. This section introduces mrgsolve, linpk, and nlme — each filling a distinct niche in the PK/PD toolkit.

4.9.1 mrgsolve: C++ Simulation for Clinical Trial Design

mrgsolve compiles ODE models to C++ for blazing-fast simulation. It excels at large-scale clinical trial simulations with complex dosing regimens.

Let’s simulate a two-compartment IV bolus in both rxode2 and mrgsolve to compare their syntax and output:

Code 
library(rxode2)
library(mrgsolve)
library(ggplot2)
library(dplyr)

# ---- Simulated patient parameters ----
params <- c(CL = 5, Vc = 50, Q = 4, Vp = 30)  # named numeric vector

# ---- rxode2 model ----
rx_model <- rxode2({
  d/dt(A1) <- - (CL/Vc) * A1 - (Q/Vc) * A1 + (Q/Vp) * A2
  d/dt(A2) <-   (Q/Vc) * A1 - (Q/Vp) * A2
  Cp <- A1 / Vc
})

rx_sim <- rx_model |>
  rxSolve(
    params  = params,
    events  = et(time = 0, amt = 500, cmt = 1) |> et(seq(0, 48, by = 0.5))
  ) |>
  as.data.frame() |>
  dplyr::mutate(engine = "rxode2")

# ---- mrgsolve model ----
mrg_code <- "
$PARAM CL = 5, Vc = 50, Q = 4, Vp = 30
$CMT A1 A2
$ODE
dxdt_A1 = - (CL/Vc) * A1 - (Q/Vc) * A1 + (Q/Vp) * A2;
dxdt_A2 =   (Q/Vc) * A1 - (Q/Vp) * A2;
$TABLE
double Cp = A1 / Vc;
$CAPTURE Cp
"

mrg_model <- mcode("twocmt", mrg_code)
mrg_sim <- mrg_model |>
  ev(amt = 500, cmt = 1, time = 0) |>
  mrgsim(end = 48, delta = 0.5) |>
  as.data.frame() |>
  mutate(engine = "mrgsolve")

# ---- Combine & Plot ----
bind_rows(rx_sim, mrg_sim) |>
  ggplot(aes(x = time, y = Cp, color = engine, linetype = engine)) +
  geom_line(linewidth = 1.0) +
  scale_color_manual(values = c("rxode2" = "#00A8B0", "mrgsolve" = "#E0533D")) +
  scale_linetype_manual(values = c("rxode2" = "solid", "mrgsolve" = "dashed")) +
  scale_y_log10() +
  labs(
    title = "rxode2 vs mrgsolve: 2-Compartment IV Bolus",
    subtitle = "Both engines converge to identical solutions",
    x = "Time (h)", y = "Concentration (mg/L)",
    color = "Engine", linetype = "Engine"
  ) +
  theme_minimal(base_size = 13)

The teal (rxode2, solid) and coral (mrgsolve, dashed) curves sit exactly on top of each other — confirming that both engines produce identical numerical solutions. The two phases (rapid distribution followed by slower elimination) are clearly visible on the log scale. In practice, choose the engine that fits your workflow: rxode2 for nlmixr2 integration, mrgsolve for large-scale clinical trial simulations.

Feature rxode2 mrgsolve
Syntax R-like ODE in rxode2({...}) C++ code block $ODE ...
Compilation C (via Rcpp) C++ (via Rcpp)
Dosing et() event table ev() event object
Population sim rxSolve() with parameter vectors idata_set with individual parameters
Strengths Seamless nlmixr2 integration Fastest for large-scale trial simulations

4.9.2 linpk: Closed-Form PK Without ODEs

For linear PK systems, you often don’t need an ODE solver at all. linpk computes PK profiles analytically from primary parameters, making it ideal for rapid exploration.

Code 
library(linpk)

# Single oral dose, 1-compartment
pk1 <- pkprofile(
  time = seq(0, 72, by = 0.25),
  dose = list(oral = data.frame(time = 0, amount = 100)),
  cl   = 5, vc = 80, ka = 0.8
)

as.data.frame(pk1) |>
  ggplot(aes(x = time, y = conc)) +
  geom_area(fill = "#00A8B0", alpha = 0.15) +
  geom_line(color = "#00A8B0", linewidth = 1.0) +
  labs(
    title = "linpk: Closed-Form 1-CMT Oral Profile",
    subtitle = "No differential equation solver needed",
    x = "Time (h)", y = "Concentration (mg/L)"
  ) +
  theme_minimal(base_size = 13)

The shaded area highlights drug exposure over time (related to AUC), while the smooth curve shows the closed-form solution. Unlike ODE-based approaches, linpk computes this instantly from analytical equations — ideal for teaching, rapid dose exploration, and cases where linear PK holds.

4.9.3 nlme: Linear Mixed-Effects for PK Summary Statistics

Before diving into full population PK (Chapter 12), nlme provides a gentle introduction to mixed-effects modeling applied to PK endpoints like C_{max} and AUC:

Code 
library(nlme)

# Simulate peak concentrations from 3 dose groups
set.seed(456)
pk_summary <- data.frame(
  subject = rep(1:30, each = 2),
  period  = rep(1:2, 30),
  dose    = rep(c(50, 100, 200), each = 20),
  cmax    = NA
)
pk_summary$cmax <- with(pk_summary,
  2.5 * dose^0.85 * exp(rnorm(nrow(pk_summary), 0, 0.2)) +
  rnorm(nrow(pk_summary), 0, 0.5)
)

fit <- lme(log(cmax) ~ log(dose), random = ~ 1 | subject,

4.9.5 Common Mistakes

  • Using absolute file paths — scripts break when shared or moved
  • Not checking for BLOQ (Below Limit of Quantification) values — they need special handling
  • Plotting concentration on linear scale — always check the semi-log view
  • Forgetting to convert time units (hours → minutes, days → hours)
  • Overwriting raw data — always keep original files unchanged

4.9.6 Key Takeaway

A well-organized R environment with proper project structure, version control, and reproducible scripts is the foundation of reliable PK/PD analysis. Master data import, cleaning, and visualization in R before moving to modeling.

## Handling Missing Data in PK/PD

Missing concentration values are common in clinical PK studies. The appropriate handling strategy depends on **why** the data are missing.

### The MCAR/MAR/MNAR Framework

| Mechanism | Definition | Test/Clue | Action |
|---|---|---|---|
| **MCAR** (Missing Completely at Random) | Missingness unrelated to any variable | No pattern in missing vs observed | Mean/median imputation OK |
| **MAR** (Missing at Random) | Missingness explained by observed covariates | Pattern linked to recorded variables (e.g., more missing at later times) | Regression imputation, multiple imputation (`mice`), or MLE-based modeling |
| **MNAR** (Missing Not at Random) | Missingness depends on the unobserved value itself (e.g., samples below LLOQ) | Plausible mechanism: high CL patients drop out, BLQ samples | Selection/pattern-mixture models + sensitivity analysis; **no method is unbiased** |

```{r}
# Example: Identify and handle missing data patterns
library(dplyr)

# Simulate PK data with different missingness types
set.seed(42)
pk_missing <- data.frame(
  ID = rep(1:10, each = 8),
  TIME = rep(c(0.5, 1, 2, 4, 6, 8, 12, 24), 10),
  CONC = 25 * exp(-0.2 * rep(c(0.5, 1, 2, 4, 6, 8, 12, 24), 10)) * 
          exp(rnorm(80, 0, 0.12))
)

# Introduce MCAR: random 10% missing
mcar_idx <- sample(1:80, 8)
pk_missing$CONC_MCAR <- pk_missing$CONC
pk_missing$CONC_MCAR[mcar_idx] <- NA

# Introduce MAR: more missing at later time points
mar_prob <- 0.1 * pk_missing$TIME / 24
mar_idx <- which(runif(80) < mar_prob)
pk_missing$CONC_MAR <- pk_missing$CONC
pk_missing$CONC_MAR[mar_idx] <- NA

# Summarize missingness
missing_summary <- pk_missing %>%
  summarise(
    MCAR_missing = sum(is.na(CONC_MCAR)),
    MAR_missing  = sum(is.na(CONC_MAR)),
    .by = TIME
  )

cat("Missing by time point:\n")
print(missing_summary)

# Simple imputation: last observation carried forward (LOCF)
pk_missing <- pk_missing %>%
  group_by(ID) %>%
  tidyr::fill(CONC_MCAR, CONC_MAR, .direction = "down") %>%
  ungroup()
```

::: {.key-box}
### BLQ (Below Limit of Quantification) Handling

BLQ values are a special case of **MNAR** — the value is missing *because* it's low. Common approaches:

- **LLOQ/2 substitution** — set BLQ = LLOQ/2 (simple, common in NCA)
- **M3 method** — likelihood-based approach that treats BLQ as left-censored observations
- **Imputation** — multiple imputation with left-censoring constraint

For NCA, LLOQ/2 is standard. For population PK, likelihood-based methods (M3) are preferred as they correctly propagate uncertainty.
:::

## Beyond `rxode2`: Comparing Simulation Engines

While `rxode2` is our workhorse for ODE-based PK modeling, R offers several complementary simulation frameworks. This section introduces `mrgsolve`, `linpk`, and `nlme` — each filling a distinct niche in the PK/PD toolkit.

### `mrgsolve`: C++ Simulation for Clinical Trial Design

`mrgsolve` compiles ODE models to C++ for blazing-fast simulation. It excels at large-scale clinical trial simulations with complex dosing regimens.

Let's simulate a two-compartment IV bolus in **both** `rxode2` and `mrgsolve` to compare their syntax and output:

```{r}
#| label: fig-engine-comparison
#| fig-cap: "Two-compartment IV bolus simulated with rxode2 (solid teal) and mrgsolve (dashed coral). Both engines produce identical results, confirming model equivalence."
#| fig-width: 8
#| fig-height: 5


library(rxode2)
library(mrgsolve)
library(ggplot2)
library(dplyr)

# ---- Simulated patient parameters ----
params <- c(CL = 5, Vc = 50, Q = 4, Vp = 30)  # named numeric vector

# ---- rxode2 model ----
rx_model <- rxode2({
  d/dt(A1) <- - (CL/Vc) * A1 - (Q/Vc) * A1 + (Q/Vp) * A2
  d/dt(A2) <-   (Q/Vc) * A1 - (Q/Vp) * A2
  Cp <- A1 / Vc
})

rx_sim <- rx_model |>
  rxSolve(
    params  = params,
    events  = et(time = 0, amt = 500, cmt = 1) |> et(seq(0, 48, by = 0.5))
  ) |>
  as.data.frame() |>
  dplyr::mutate(engine = "rxode2")

# ---- mrgsolve model ----
mrg_code <- "
$PARAM CL = 5, Vc = 50, Q = 4, Vp = 30
$CMT A1 A2
$ODE
dxdt_A1 = - (CL/Vc) * A1 - (Q/Vc) * A1 + (Q/Vp) * A2;
dxdt_A2 =   (Q/Vc) * A1 - (Q/Vp) * A2;
$TABLE
double Cp = A1 / Vc;
$CAPTURE Cp
"

mrg_model <- mcode("twocmt", mrg_code)
mrg_sim <- mrg_model |>
  ev(amt = 500, cmt = 1, time = 0) |>
  mrgsim(end = 48, delta = 0.5) |>
  as.data.frame() |>
  mutate(engine = "mrgsolve")

# ---- Combine & Plot ----
bind_rows(rx_sim, mrg_sim) |>
  ggplot(aes(x = time, y = Cp, color = engine, linetype = engine)) +
  geom_line(linewidth = 1.0) +
  scale_color_manual(values = c("rxode2" = "#00A8B0", "mrgsolve" = "#E0533D")) +
  scale_linetype_manual(values = c("rxode2" = "solid", "mrgsolve" = "dashed")) +
  scale_y_log10() +
  labs(
    title = "rxode2 vs mrgsolve: 2-Compartment IV Bolus",
    subtitle = "Both engines converge to identical solutions",
    x = "Time (h)", y = "Concentration (mg/L)",
    color = "Engine", linetype = "Engine"
  ) +
  theme_minimal(base_size = 13)
```

The teal (rxode2, solid) and coral (mrgsolve, dashed) curves sit exactly on top of each other — confirming that both engines produce identical numerical solutions. The two phases (rapid distribution followed by slower elimination) are clearly visible on the log scale. In practice, choose the engine that fits your workflow: rxode2 for nlmixr2 integration, mrgsolve for large-scale clinical trial simulations.

| Feature | `rxode2` | `mrgsolve` |
|---|---|---|
| **Syntax** | R-like ODE in `rxode2({...})` | C++ code block `$ODE ...` |
| **Compilation** | C (via Rcpp) | C++ (via Rcpp) |
| **Dosing** | `et()` event table | `ev()` event object |
| **Population sim** | `rxSolve()` with parameter vectors | `idata_set` with individual parameters |
| **Strengths** | Seamless nlmixr2 integration | Fastest for large-scale trial simulations |

### `linpk`: Closed-Form PK Without ODEs

For linear PK systems, you often don't need an ODE solver at all. `linpk` computes PK profiles analytically from primary parameters, making it ideal for rapid exploration.

```{r}
#| label: fig-linpk-demo
#| fig-cap: "Multi-dose oral profile generated with linpk using closed-form equations. No ODE solver required."
#| fig-width: 7
#| fig-height: 4.5


library(linpk)

# Single oral dose, 1-compartment
pk1 <- pkprofile(
  time = seq(0, 72, by = 0.25),
  dose = list(oral = data.frame(time = 0, amount = 100)),
  cl   = 5, vc = 80, ka = 0.8
)

as.data.frame(pk1) |>
  ggplot(aes(x = time, y = conc)) +
  geom_area(fill = "#00A8B0", alpha = 0.15) +
  geom_line(color = "#00A8B0", linewidth = 1.0) +
  labs(
    title = "linpk: Closed-Form 1-CMT Oral Profile",
    subtitle = "No differential equation solver needed",
    x = "Time (h)", y = "Concentration (mg/L)"
  ) +
  theme_minimal(base_size = 13)
```

The shaded area highlights drug exposure over time (related to AUC), while the smooth curve shows the closed-form solution. Unlike ODE-based approaches, `linpk` computes this instantly from analytical equations — ideal for teaching, rapid dose exploration, and cases where linear PK holds.

### `nlme`: Linear Mixed-Effects for PK Summary Statistics

Before diving into full population PK (Chapter 12), `nlme` provides a gentle introduction to mixed-effects modeling applied to PK endpoints like $C_{max}$ and AUC:

```{r}
library(nlme)

# Simulate peak concentrations from 3 dose groups
set.seed(456)
pk_summary <- data.frame(
  subject = rep(1:30, each = 2),
  period  = rep(1:2, 30),
  dose    = rep(c(50, 100, 200), each = 20),
  cmax    = NA
)
pk_summary$cmax <- with(pk_summary,
  2.5 * dose^0.85 * exp(rnorm(nrow(pk_summary), 0, 0.2)) +
  rnorm(nrow(pk_summary), 0, 0.5)
)

fit <- lme(log(cmax) ~ log(dose), random = ~ 1 | subject,
           data = pk_summary)
summary(fit)
```

The power model $\text{C}_{max} \propto \text{dose}^{\beta}$ estimates $\beta \approx 0.85$, confirming near-linear dose proportionality — a common regulatory question. We'll revisit `nlme` for full PK curve fitting in Chapter 12.
