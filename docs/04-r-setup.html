4.8 Handling Missing Data in PK/PD

Missing concentration values are common in clinical PK studies. The appropriate handling strategy depends on why the data are missing.

4.8.1 The MCAR/MAR/MNAR Framework

Mechanism Definition Test/Clue Action
MCAR (Missing Completely at Random) Missingness unrelated to any variable No pattern in missing vs observed Mean/median imputation OK
MAR (Missing at Random) Missingness explained by observed covariates Pattern linked to recorded variables (e.g., more missing at later times) Regression imputation, multiple imputation (mice), or MLE-based modeling
MNAR (Missing Not at Random) Missingness depends on the unobserved value itself (e.g., samples below LLOQ) Plausible mechanism: high CL patients drop out, BLQ samples Selection/pattern-mixture models + sensitivity analysis; no method is unbiased
Code 
# Example: Identify and handle missing data patterns
library(dplyr)

# Simulate PK data with different missingness types
set.seed(42)
pk_missing <- data.frame(
  ID = rep(1:10, each = 8),
  TIME = rep(c(0.5, 1, 2, 4, 6, 8, 12, 24), 10),
  CONC = 25 * exp(-0.2 * rep(c(0.5, 1, 2, 4, 6, 8, 12, 24), 10)) * 
          exp(rnorm(80, 0, 0.12))
)

# Introduce MCAR: random 10% missing
mcar_idx <- sample(1:80, 8)
pk_missing$CONC_MCAR <- pk_missing$CONC
pk_missing$CONC_MCAR[mcar_idx] <- NA

# Introduce MAR: more missing at later time points
mar_prob <- 0.1 * pk_missing$TIME / 24
mar_idx <- which(runif(80) < mar_prob)
pk_missing$CONC_MAR <- pk_missing$CONC
pk_missing$CONC_MAR[mar_idx] <- NA

# Summarize missingness
missing_summary <- pk_missing %>%
  summarise(
    MCAR_missing = sum(is.na(CONC_MCAR)),
    MAR_missing  = sum(is.na(CONC_MAR)),
    .by = TIME
  )

4.8.2 BLQ (Below Limit of Quantification) Handling

BLQ values are a special case of MNAR — the value is missing because it’s low. Common approaches:

  • LLOQ/2 substitution — set BLQ = LLOQ/2 (simple, common in NCA)
  • M3 method — likelihood-based approach that treats BLQ as left-censored observations
  • Imputation — multiple imputation with left-censoring constraint

For NCA, LLOQ/2 is standard. For population PK, likelihood-based methods (M3) are preferred as they correctly propagate uncertainty.

4.9 Beyond rxode2: Comparing Simulation Engines

While rxode2 is our workhorse for ODE-based PK modeling, R offers several complementary simulation frameworks. This section introduces mrgsolve, linpk, and nlme — each filling a distinct niche in the PK/PD toolkit.

4.9.1 mrgsolve: C++ Simulation for Clinical Trial Design

mrgsolve compiles ODE models to C++ for blazing-fast simulation. It excels at large-scale clinical trial simulations with complex dosing regimens.

Let’s simulate a two-compartment IV bolus in both rxode2 and mrgsolve to compare their syntax and output:

Code 
library(rxode2)
library(mrgsolve)
library(ggplot2)
library(dplyr)

# ---- Simulated patient parameters ----
params <- c(CL = 5, Vc = 50, Q = 4, Vp = 30)  # named numeric vector

# ---- rxode2 model ----
rx_model <- rxode2({
  d/dt(A1) <- - (CL/Vc) * A1 - (Q/Vc) * A1 + (Q/Vp) * A2
  d/dt(A2) <-   (Q/Vc) * A1 - (Q/Vp) * A2
  Cp <- A1 / Vc
})

rx_sim <- rx_model |>
  rxSolve(
    params  = params,
    events  = et(time = 0, amt = 500, cmt = 1) |> et(seq(0, 48, by = 0.5))
  ) |>
  as.data.frame() |>
  dplyr::mutate(engine = "rxode2")

# ---- mrgsolve model ----
mrg_code <- "
$PARAM CL = 5, Vc = 50, Q = 4, Vp = 30
$CMT A1 A2
$ODE
dxdt_A1 = - (CL/Vc) * A1 - (Q/Vc) * A1 + (Q/Vp) * A2;
dxdt_A2 =   (Q/Vc) * A1 - (Q/Vp) * A2;
$TABLE
double Cp = A1 / Vc;
$CAPTURE Cp
"

mrg_model <- mcode("twocmt", mrg_code)
mrg_sim <- mrg_model |>
  ev(amt = 500, cmt = 1, time = 0) |>
  mrgsim(end = 48, delta = 0.5) |>
  as.data.frame() |>
  mutate(engine = "mrgsolve")

# ---- Combine & Plot ----
bind_rows(rx_sim, mrg_sim) |>
  ggplot(aes(x = time, y = Cp, color = engine, linetype = engine)) +
  geom_line(linewidth = 1.0) +
  scale_color_manual(values = c("rxode2" = "#00A8B0", "mrgsolve" = "#E0533D")) +
  scale_linetype_manual(values = c("rxode2" = "solid", "mrgsolve" = "dashed")) +
  scale_y_log10() +
  labs(
    title = "rxode2 vs mrgsolve: 2-Compartment IV Bolus",
    subtitle = "Both engines converge to identical solutions",
    x = "Time (h)", y = "Concentration (mg/L)",
    color = "Engine", linetype = "Engine"
  ) +
  theme_minimal(base_size = 13)

5 The One-Compartment Model

5.1 Learning Objectives

By the end of this chapter you will be able to:

  • Estimate PK parameters from concentration-time data using nonlinear regression
  • Understand and identify flip-flop kinetics

5.2 What Is a Compartment?

A compartment is a mathematical abstraction representing a kinetically homogeneous pool of drug in the body. It does NOT correspond to a specific anatomical space — it is defined by the rate at which drug equilibrates.

In a one-compartment model, we assume the drug distributes instantaneously and uniformly throughout the body.

5.2.1 Core Mental Models

  1. The Null Model: Always fit the one-compartment model first as a baseline. Only escalate to multi-compartment (Ch 7) if residuals show systematic biphasic curvature.

  2. The CL-V-ke Triad: These three parameters are interchangeable — know any two and derive the third via CL = k_e \cdot V and t_{1/2} = \ln(2)/k_e. This means the model has only two independent parameters.

  3. Oral C(t) = Difference of Exponentials: For first-order absorption, the concentration-time curve is C(t) = \frac{F \cdot D \cdot k_a}{V(k_a - k_e)}(e^{-k_e t} - e^{-k_a t}) — whichever rate constant is smaller (the “flip-flop” condition) dominates the terminal slope.

  4. AUC = Dose/CL quick sanity check: For IV bolus, AUC is independent of the full time-course — compute it directly as Dose/CL to validate simulation output.

5.3 IV Bolus: The Simplest Model

5.3.1 Model Equations

For a single IV bolus dose:

\frac{dA}{dt} = -k_e \cdot A

C(t) = \frac{D}{V} \cdot e^{-k_e t} = C_0 \cdot e^{-k_e t}

where C_0 is the concentration at t = 0 (immediately after injection).

library(rxode2)
library(dplyr)
library(ggplot2)

# Define the one-compartment IV bolus model
mod_iv_bolus <- rxode2({
  d/dt(central) <- -ke * central
  conc <- central / V
})

# Parameters
params <- c(ke = 0.3, V = 10)  # ke = 0.3 1/h, V = 10 L
dose_amount <- 100  # mg

5.3.2 Route-Specific Solutions

Route C(t) Formula Notes
IV bolus C(t) = \frac{D}{V}e^{-k_e t} F = 1, instantaneous
Oral (1st-order absorption) C(t) = \frac{F D k_a}{V(k_a - k_e)}(e^{-k_e t} - e^{-k_a t}) F < 1 typically
Constant infusion C(t) = \frac{R}{k_e V}(1 - e^{-k_e t}) + C_0 e^{-k_e t} Approaches steady state R/(k_e V)

5.3.3 Core Parameter Relationships

Parameter Formula Note
CL k_e \cdot V Clearance (volume/time)
V D / C(0) Volume of distribution
t_{1/2} \ln(2)/k_e Half-life
AUC (IV) Dose/CL Quick sanity check independent of time-course

5.3.4 Worked Example: Manual Calculation

Bolus IV dose: D = 500 mg, V = 50 L, k_e = 0.2 h^{-1}.

C(t) = \frac{500}{50} \cdot e^{-0.2t} = 10 \cdot e^{-0.2t} \text{ mg/L}

  • C(0) = 10 mg/L
  • t_{1/2} = \ln(2)/0.2 \approx 3.47 h \rightarrow C(3.47) = 5 mg/L (verifies half-life)
  • CL = k_e \cdot V = 0.2 \cdot 50 = 10 L/h
  • AUC = Dose/CL = 500/10 = 50 mg·h/L

5.4 IV Infusion

5.4.1 Model Equations

For a constant-rate IV infusion:

\frac{dA}{dt} = R - k_e \cdot A

where R is the infusion rate (amount/time).

At steady state: C_{ss} = \frac{R}{CL} = \frac{R}{k_e \cdot V}

# Define the IV infusion model

  d/dt(central) <- rate - ke * central

5.5 Oral Dosing (First-Order Absorption)

5.5.1 Model Equations

For oral dosing with first-order absorption:

\frac{dA_{gut}}{dt} = -k_a \cdot A_{gut}

\frac{dA}{dt} = k_a \cdot A_{gut} - k_e \cdot A

C(t) = \frac{F \cdot D \cdot k_a}{V \cdot (k_a - k_e)} \cdot (e^{-k_e t} - e^{-k_a t})

# Define the one-compartment oral model

5.9 Alternative: ODE-Based Simulation with deSolve

While we use rxode2 throughout this book, the same model can be implemented with the deSolve package for readers who prefer it:


library(deSolve)

# Single-compartment ODE with optional constant infusion input
pk_ode <- function(time, state, parameters) {
  C <- state[1]
  k_e <- parameters["k_e"]
  V <- parameters["V"]
  input_rate <- parameters["RateIn"]
  dCdt <- -k_e * C + input_rate / V
  list(c(dCdt))
}

simulate_pk <- function(k_e, V, Dose, RateIn, t_max, dt, route = "IV-bolus") {
  C0 <- if (route == "IV-bolus") Dose / V else if (route == "Infusion") 0 else
    stop("Unsupported route. Choose 'IV-bolus' or 'Infusion'.")
  ode(y = c(C = C0), times = seq(0, t_max, by = dt), func = pk_ode,
      parms = c(k_e = k_e, V = V, RateIn = RateIn)) |> as.data.frame()
}

calc_half_life <- function(k_e) log(2) / k_e
calc_clearance <- function(k_e, V) k_e * V
calc_auc       <- function(Dose, CL) Dose / CL

This demonstrates route-aware initial conditions (C_0 = Dose/V for bolus vs 0 for infusion) feeding a single shared ODE.

5.9.1 Common Mistakes

  • Assuming k_a > k_e always — always verify with IV data
  • Not checking for flip-flop: oral-only data cannot distinguish k_a from k_e without IV reference
  • Using linear scale for terminal slope — use semi-log
  • Ignoring the absorption lag time — some oral formulations have a delay before absorption begins
  • Fitting too many parameters with too few data points
  • Ignoring tissue/protein binding when V is unexpectedly large or small: the one-compartment model assumes no significant binding alters free drug concentration. Large discrepancies in V often signal binding effects better handled in Ch 11.
  • Fitting k_e and V via nls with poor initial guesses: nonlinear least squares can fail to converge or find local minima. Use nls.control(maxiter = 500) and reasonable starting values. Always verify against the analytical formulas.

5.9.2 Key Takeaway

The one-compartment model is the foundation of all PK modeling. Key principles:

  1. One ODE, many routes: dC/dt = -k_e C + D(t)/V — the master equation; D(t) = 0 for bolus, constant for infusion, involves k_a for oral.
  2. CL-V-ke are interchangeable: CL = k_e \cdot V, t_{1/2} = \ln(2)/k_e — any two give the third.
  3. AUC = Dose/CL for IV is a fast independent check on simulation correctness.
  4. Always fit one-compartment first as a baseline before adding compartments.
  5. Check for flip-flop when analyzing oral data without IV reference.

5.11.2 Micro-Macro Parameter Translation

PK parameters come in two flavors. Micro-constants (k_{e}, k_{10}, k_{12}, k_{21}) describe rates between compartments. Macro-constants (CL, V, Q, Vp) are more clinically intuitive. The conversion is straightforward:

# 1-compartment: macro → micro
# CL = ke * V   →   ke = CL / V
CL <- 5; V <- 50
ke <- CL / V
cat("ke = CL / V =", ke, "1/h\n")

# 1-compartment: micro → macro
# V = Dose / C0,  CL = ke * V
C0 <- 10; ke <- 0.1
V  <- 100 / C0
CL <- ke * V
cat("V =", V, "L, CL =", CL, "L/h\n")

For 2-compartment models, the PKconverter package automates these translations using TwoComp_Volume_Clearance() and TwoComp_Volume_RateConstant().

This is especially useful when reading literature — papers often report one form or the other.

6.2 What Is Pharmacodynamics?

Pharmacodynamics describes the relationship between drug concentration (or dose) and the magnitude of the pharmacological effect. The fundamental question is:

Given a certain drug concentration at the site of action, how large is the effect?

6.3 The Emax Model

6.3.1 Model Equation

E = E_0 + \frac{E_{max} \cdot C}{EC_{50} + C}

where: - E: Observed effect - E_0: Baseline effect (at C = 0) - E_{max}: Maximum drug-attributable effect - EC_{50}: Concentration producing 50% of E_{max} - C: Drug concentration

6.8 Receptor Theory: The Mechanistic Basis of PD Curves

The empirical Emax and sigmoid Emax models describe concentration-effect data well, but they don’t explain why the curves have that shape. Receptor theory bridges this gap by deriving PD curves from the fundamental chemistry of drug-receptor binding.

6.8.1 Mass-Action Binding

The simplest model: a drug molecule (L) binds reversibly to a receptor (R):

R + L \underset{k_{-1}}{\stackrel{k_1}{\rightleftharpoons}} RL

At equilibrium, the dissociation constant K_d is:

K_d = \frac{k_{-1}}{k_1} = \frac{[R][L]}{[RL]}

The fraction of receptors occupied (occupancy, \theta) follows the Langmuir isotherm:

\theta = \frac{[RL]}{[R]_{total}} = \frac{[L]}{K_d + [L]}

This is exactly the same functional form as the Emax model — the Emax curve arises naturally from simple 1:1 binding. EC_{50} corresponds to K_d, and E_{max} corresponds to the maximal response when all receptors are occupied.

# Demonstrate receptor occupancy as a function of concentration
L <- 10^seq(-2, 3, length.out = 200)
Kd <- 3.5

# Occupancy (Langmuir isotherm) vs Emax model
occupancy <- L / (Kd + L)
emax_equiv <- L / (Kd + L) * 100  # scaled to 100%

df_receptor <- data.frame(
  Concentration = rep(L, 2),
  Value = c(occupancy, emax_equiv / 100),
  Curve = rep(c("Occupancy (θ)", "Emax (scaled)"), each = length(L))
)

ggplot(df_receptor, aes(Concentration, Value, color = Curve)) +

6.8.2 From Occupancy to Effect: The Operational Model of Agonism

Not all occupied receptors produce the same effect. The operational model of agonism (Black & Leff, 1983) accounts for the transduction efficiency between binding and response:

E = \frac{E_{max} \cdot \theta^n}{\theta^n + K_A^n}

Substituting \theta = [L]/(K_d + [L]) gives a function that reduces to the sigmoid Emax model under most conditions. This explains why the Hill coefficient (\gamma) can differ from 1 even with simple 1:1 binding — it reflects signal amplification downstream of the receptor.

6.8.3 The Chain: Binding → Occupancy → Effect

\text{Mass action: } R+L \rightleftharpoons RL \quad\rightarrow\quad \theta = \frac{[L]}{K_d+[L]}

\text{Operational model: } E = E_{max} \cdot \frac{\theta^n}{\theta^n + K_A^n}

\text{Empirical (Hill): } E = E_{max} \cdot \frac{[L]^\gamma}{EC_{50}^\gamma + [L]^\gamma}

These three equations form a nested hierarchy — each is a special case or generalization of the one above.

6.9 Agonism, Antagonism, and Allosteric Modulation

6.9.1 Agonists

An agonist binds to a receptor and produces a response. The potency is reflected in EC_{50} (or K_d), and the efficacy in E_{max}.

6.9.2 Competitive Antagonism

A competitive antagonist binds to the same site as the agonist but produces no response. It shifts the dose-response curve to the right without reducing E_{max}:

E = E_{max} \cdot \frac{[A]}{[A] + EC_{50} \cdot (1 + [B]/K_B)}

where [A] is agonist concentration, [B] is antagonist concentration, and K_B is the antagonist’s dissociation constant.

# Competitive antagonism: rightward shift without Emax reduction
A <- 10^seq(-2, 3, length.out = 200)
EC50 <- 8; Emax <- 100
B_conc <- c(0, 2, 8, 25)  # antagonist concentrations
KB <- 4                      # antagonist Kd

df_antag <- expand.grid(A = A, B = B_conc) %>%
  mutate(
    EC50_app = EC50 * (1 + B / KB),
    Effect = Emax * A / (EC50_app + A),
    Antagonist = factor(paste0("[B] = ", B))
  )

ggplot(df_antag, aes(A, Effect, color = Antagonist)) +
  geom_line(linewidth = 1) +
  scale_x_log10() +
  labs(title = "Competitive Antagonism",
       subtitle = "Rightward shift in dose-response curve; Emax preserved",
       x = "[Agonist] (log scale)", y = "Effect (% of maximum)") +
  scale_color_manual(values = c("#0B6E6B", "#D4A843", "#C44536", "#1B365D")) +
  theme_minimal()

6.9.3 Non-Competitive Antagonism

A non-competitive antagonist binds to an allosteric site and reduces the maximal response without shifting EC_{50} (in the pure case), or both reduces E_{max} and shifts EC_{50}:

E = \frac{E_{max}}{1 + [B]/K_B} \cdot \frac{[A]}{[A] + EC_{50}}

6.9.4 Allosteric Modulation

Allosteric modulators bind at a site distinct from the orthosteric (agonist) site and change receptor affinity (\alpha) and/or efficacy (\beta):

  • Positive allosteric modulator (PAM): increases agonist affinity or efficacy
  • Negative allosteric modulator (NAM): decreases agonist affinity or efficacy

6.9.5 Clinical Significance

Understanding antagonism mechanisms is essential for: - Reversal agents: Naloxone (competitive antagonist) reverses opioid overdose - DDI prediction: Co-administered antagonists shift the effective dose - Therapeutic window: Irreversible antagonists can cause prolonged effects

6.10.2 Common Mistakes

  • Using linear models when the drug approaches E_{max} — linear extrapolation overestimates effect
  • Ignoring baseline (E_0): Always measure and model the baseline
  • Confusing EC_{50} with potency: Lower EC_{50} means more potent
  • Forcing \gamma = 1 without checking: Test whether the sigmoid model improves fit
  • Extrapolating far beyond the observed concentration range

## Handling Missing Data in PK/PD

Missing concentration values are common in clinical PK studies. The appropriate handling strategy depends on **why** the data are missing.

### The MCAR/MAR/MNAR Framework

| Mechanism | Definition | Test/Clue | Action |
|---|---|---|---|
| **MCAR** (Missing Completely at Random) | Missingness unrelated to any variable | No pattern in missing vs observed | Mean/median imputation OK |
| **MAR** (Missing at Random) | Missingness explained by observed covariates | Pattern linked to recorded variables (e.g., more missing at later times) | Regression imputation, multiple imputation (`mice`), or MLE-based modeling |
| **MNAR** (Missing Not at Random) | Missingness depends on the unobserved value itself (e.g., samples below LLOQ) | Plausible mechanism: high CL patients drop out, BLQ samples | Selection/pattern-mixture models + sensitivity analysis; **no method is unbiased** |

```{r}
# Example: Identify and handle missing data patterns
library(dplyr)

# Simulate PK data with different missingness types
set.seed(42)
pk_missing <- data.frame(
  ID = rep(1:10, each = 8),
  TIME = rep(c(0.5, 1, 2, 4, 6, 8, 12, 24), 10),
  CONC = 25 * exp(-0.2 * rep(c(0.5, 1, 2, 4, 6, 8, 12, 24), 10)) * 
          exp(rnorm(80, 0, 0.12))
)

# Introduce MCAR: random 10% missing
mcar_idx <- sample(1:80, 8)
pk_missing$CONC_MCAR <- pk_missing$CONC
pk_missing$CONC_MCAR[mcar_idx] <- NA

# Introduce MAR: more missing at later time points
mar_prob <- 0.1 * pk_missing$TIME / 24
mar_idx <- which(runif(80) < mar_prob)
pk_missing$CONC_MAR <- pk_missing$CONC
pk_missing$CONC_MAR[mar_idx] <- NA

# Summarize missingness
missing_summary <- pk_missing %>%
  summarise(
    MCAR_missing = sum(is.na(CONC_MCAR)),
    MAR_missing  = sum(is.na(CONC_MAR)),
    .by = TIME
  )

cat("Missing by time point:\n")
print(missing_summary)

# Simple imputation: last observation carried forward (LOCF)
pk_missing <- pk_missing %>%
  group_by(ID) %>%
  tidyr::fill(CONC_MCAR, CONC_MAR, .direction = "down") %>%
  ungroup()
```

::: {.key-box}
### BLQ (Below Limit of Quantification) Handling

BLQ values are a special case of **MNAR** — the value is missing *because* it's low. Common approaches:

- **LLOQ/2 substitution** — set BLQ = LLOQ/2 (simple, common in NCA)
- **M3 method** — likelihood-based approach that treats BLQ as left-censored observations
- **Imputation** — multiple imputation with left-censoring constraint

For NCA, LLOQ/2 is standard. For population PK, likelihood-based methods (M3) are preferred as they correctly propagate uncertainty.

### `mrgsolve`: C++ Simulation for Clinical Trial Design

`mrgsolve` compiles ODE models to C++ for blazing-fast simulation. It excels at large-scale clinical trial simulations with complex dosing regimens.

Let's simulate a two-compartment IV bolus in **both** `rxode2` and `mrgsolve` to compare their syntax and output:

```{r}
#| label: fig-engine-comparison
#| fig-cap: "Two-compartment IV bolus simulated with rxode2 (solid teal) and mrgsolve (dashed coral). Both engines produce identical results, confirming model equivalence."
#| fig-width: 8
#| fig-height: 5


library(rxode2)
library(mrgsolve)
library(ggplot2)
library(dplyr)

# ---- Simulated patient parameters ----
params <- c(CL = 5, Vc = 50, Q = 4, Vp = 30)  # named numeric vector

# ---- rxode2 model ----
rx_model <- rxode2({
  d/dt(A1) <- - (CL/Vc) * A1 - (Q/Vc) * A1 + (Q/Vp) * A2
  d/dt(A2) <-   (Q/Vc) * A1 - (Q/Vp) * A2
  Cp <- A1 / Vc
})

rx_sim <- rx_model |>
  rxSolve(
    params  = params,
    events  = et(time = 0, amt = 500, cmt = 1) |> et(seq(0, 48, by = 0.5))
  ) |>
  as.data.frame() |>
  dplyr::mutate(engine = "rxode2")

# ---- mrgsolve model ----
mrg_code <- "
$PARAM CL = 5, Vc = 50, Q = 4, Vp = 30
$CMT A1 A2
$ODE
dxdt_A1 = - (CL/Vc) * A1 - (Q/Vc) * A1 + (Q/Vp) * A2;
dxdt_A2 =   (Q/Vc) * A1 - (Q/Vp) * A2;
$TABLE
double Cp = A1 / Vc;
$CAPTURE Cp
"

mrg_model <- mcode("twocmt", mrg_code)
mrg_sim <- mrg_model |>
  ev(amt = 500, cmt = 1, time = 0) |>
  mrgsim(end = 48, delta = 0.5) |>
  as.data.frame() |>
  mutate(engine = "mrgsolve")

# ---- Combine & Plot ----
bind_rows(rx_sim, mrg_sim) |>
  ggplot(aes(x = time, y = Cp, color = engine, linetype = engine)) +
  geom_line(linewidth = 1.0) +
  scale_color_manual(values = c("rxode2" = "#00A8B0", "mrgsolve" = "#E0533D")) +
  scale_linetype_manual(values = c("rxode2" = "solid", "mrgsolve" = "dashed")) +
  scale_y_log10() +
  labs(
    title = "rxode2 vs mrgsolve: 2-Compartment IV Bolus",
    subtitle = "Both engines converge to identical solutions",
    x = "Time (h)", y = "Concentration (mg/L)",
    color = "Engine", linetype = "Engine"
  ) +
  theme_minimal(base_size = 13)
```

| Feature | `rxode2` | `mrgsolve` |
|---|---|---|
| **Syntax** | R-like ODE in `rxode2({...})` | C++ code block `$ODE ...` |
| **Compilation** | C (via Rcpp) | C++ (via Rcpp) |
| **Dosing** | `et()` event table | `ev()` event object |
| **Population sim** | `rxSolve()` with parameter vectors | `idata_set` with individual parameters |
| **Strengths** | Seamless nlmixr2 integration | Fastest for large-scale trial simulations |

### `nlme`: Linear Mixed-Effects for PK Summary Statistics

Before diving into full population PK (Chapter 12), `nlme` provides a gentle introduction to mixed-effects modeling applied to PK endpoints like $C_{max}$ and AUC:

```{r}
library(nlme)

# Simulate peak concentrations from 3 dose groups
set.seed(456)
pk_summary <- data.frame(
  subject = rep(1:30, each = 2),
  period  = rep(1:2, 30),
  dose    = rep(c(50, 100, 200), each = 20),
  cmax    = NA
)
pk_summary$cmax <- with(pk_summary,
  2.5 * dose^0.85 * exp(rnorm(nrow(pk_summary), 0, 0.2)) +
  rnorm(nrow(pk_summary), 0, 0.5)
)

fit <- lme(log(cmax) ~ log(dose), random = ~ 1 | subject,
           data = pk_summary)
summary(fit)
```

The power model $\text{C}_{max} \propto \text{dose}^{\beta}$ estimates $\beta \approx 0.85$, confirming near-linear dose proportionality — a common regulatory question. We'll revisit `nlme` for full PK curve fitting in Chapter 12.

- **Not checking for BLOQ (Below Limit of Quantification) values** — they need special handling
- **Plotting concentration on linear scale** — always check the semi-log view
- **Forgetting to convert time units** (hours → minutes, days → hours)
