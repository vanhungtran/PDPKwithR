Introduction

Phase I oncology trials aim to find the Maximum Tolerated Dose (MTD) — the highest dose with acceptable toxicity. This appendix covers three Bayesian designs: Continual Reassessment Method (CRM), Modified Toxicity Probability Interval (mTPI), and Bayesian Optimal Interval (BOIN).

Simulated Phase I Oncology Trial

A 5-dose study targeting a 25% dose-limiting toxicity (DLT) rate:

Code 
library(escalation)
library(ggplot2)
library(dplyr)
library(tidyr)

# True DLT probabilities (5 doses)
true_tox <- c(0.05, 0.12, 0.25, 0.40, 0.55)

# ---- mTPI Design ----
mtpi_design <- get_mtpi(
  num_doses            = 5,
  target               = 0.25,
  epsilon1             = 0.05,
  epsilon2             = 0.05,
  exclusion_certainty  = 0.95
) |>
  stop_at_n(n = 30)

# Show dose paths for first 2 cohorts
paths <- mtpi_design |>
  get_dose_paths(cohort_sizes = c(3, 3), next_dose = 1)

# ---- BOIN Design ----
boin_design <- get_boin(
  num_doses  = 5,
  target     = 0.25
) |>
  stop_at_n(n = 30)

# ---- CRM Design ----
crm_design <- get_dfcrm(
  skeleton = c(0.05, 0.12, 0.25, 0.40, 0.55),
  target   = 0.25
) |>
  stop_at_n(n = 30)

# ---- Simulate 200 trials for each design ----
set.seed(1122)
n_sims <- 200

sim_mtpi <- mtpi_design |>
  simulate_trials(
    num_sims      = n_sims,
    true_prob_tox = true_tox,
    next_dose     = 1
  )

sim_boin <- boin_design |>
  simulate_trials(
    num_sims      = n_sims,
    true_prob_tox = true_tox,
    next_dose     = 1
  )

sim_crm <- crm_design |>
  simulate_trials(
    num_sims      = n_sims,
    true_prob_tox = true_tox,
    next_dose     = 1
  )

# ---- Operating characteristics ----
oc_plot <- data.frame(
  Design = rep(c("mTPI", "BOIN", "CRM"), each = 5),
  Dose   = rep(1:5, 3),
  P_Rec  = c(
    prob_recommend(sim_mtpi)[1:5],
    prob_recommend(sim_boin)[1:5],
    prob_recommend(sim_crm)[1:5]
  )
)

ggplot(oc_plot, aes(x = factor(Dose), y = P_Rec * 100, fill = Design)) +
  geom_col(position = "dodge", alpha = 0.85) +
  geom_hline(yintercept = 0, linewidth = 0.5) +
  scale_fill_manual(values = c("mTPI" = "#00A8B0", "BOIN" = "#E0533D",
                                "CRM" = "#6C63B5")) +
  labs(
    title = "Correct MTD Selection: Three Dose-Finding Designs",
    subtitle = paste("True MTD = Dose 3 (25% DLT).", n_sims, "simulations each."),
    x = "Dose Level", y = "Probability of Recommendation (%)",
    fill = "Design"
  ) +
  theme_minimal(base_size = 13)

All three designs correctly select Dose 3 most often. mTPI and BOIN have similar performance; CRM is slightly more efficient but requires careful skeleton specification.

Individual Trial Simulation: mTPI

Code 
# Single trial walk-through
set.seed(42)
single_trial <- mtpi_design |>
  simulate_trials(
    num_sims      = 1,
    true_prob_tox = true_tox,
    next_dose     = 1
  )

# Extract dose decisions
decisions <- single_trial$dose_indices[[1]]
cat("Dose sequence:", paste(decisions, collapse = " → "), "\n")
cat("Final MTD recommendation:", single_trial$recommended_dose[1], "\n")
Design Strengths Limitations
mTPI Simple interval-based; well-studied Requires epsilon tuning
BOIN Pre-specified escalation rules; easy to implement Less flexible than CRM
CRM Model-based; uses all data efficiently Skeleton sensitivity; complex

Key Takeaway

Phase I dose-finding is a distinct discipline bridging PK/PD and clinical trial design. The escalation package unifies CRM, mTPI, and BOIN under a common interface with simulation-based operating characteristic assessment.

## Design Comparison

```{r}
#| label: tbl-design-comparison
#| fig-cap: "Comparison of Phase I dose-finding designs"
#| eval: false

comp_table <- data.frame(
  Feature = c("Method", "Target specification", "Dose-skipping",
              "Overdose control", "Simplicity"),
  mTPI    = c("Bayesian UPM intervals", "Target + epsilon bounds",
              "Optional rule", "Exclusion probability > xi", "Moderate"),
  BOIN    = c("Bayesian optimal interval", "Target only",
              "Optional rule", "Escalation boundary", "High"),
  CRM     = c("Bayesian model-based", "Target via skeleton",
              "Model-driven", "Dose-centered skeleton", "Low (needs calibration)")
)

knitr::kable(comp_table, caption = "Design comparison for Phase I dose-finding")
```

---

| Design | Strengths | Limitations |
|---|---|---|
| **mTPI** | Simple interval-based; well-studied | Requires epsilon tuning |
| **BOIN** | Pre-specified escalation rules; easy to implement | Less flexible than CRM |
| **CRM** | Model-based; uses all data efficiently | Skeleton sensitivity; complex |

::: {.key-box}
### Key Takeaway

Phase I dose-finding is a distinct discipline bridging PK/PD and clinical trial design. The `escalation` package unifies CRM, mTPI, and BOIN under a common interface with simulation-based operating characteristic assessment.
